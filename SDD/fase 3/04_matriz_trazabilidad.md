# Matriz de Trazabilidad

## Objetivo

Relacionar vision de producto, historias de usuario, modulos tecnicos y tareas del MVP para asegurar que el trabajo implementado preserve la intencion original.

| Vision / Capacidad | Historia de usuario | Modulo tecnico | Tareas relacionadas |
|---|---|---|---|
| Acceso seguro al producto | HU-01 | Identidad y acceso | T3-001, T3-002, T3-003, T3-004 |
| Registro de ingresos y gastos | HU-02 | Transacciones | T3-009, T3-010, T3-011, T3-012, T3-013 |
| Clasificacion de movimientos | HU-03 | Categorias | T3-007, T3-008 |
| Representacion de saldos | HU-02, HU-10 | Cuentas y saldos | T3-005, T3-006, T3-012, T3-016 |
| Informes financieros basicos | HU-05 | Reportes basicos | T3-014, T3-015 |
| Resumen inicial del usuario | HU-10 | Dashboard | T3-016 |
| Estabilidad de experiencia | Transversal | Calidad minima | T3-017, T3-018 |

## Cobertura del MVP

### Capacidades cubiertas

- acceso autenticado,
- cuentas financieras basicas,
- categorias iniciales,
- registro de transacciones,
- actualizacion de saldos,
- consulta y resumen mensual,
- panel basico.

### Capacidades diferidas

- presupuestos completos,
- metas de ahorro,
- notificaciones avanzadas,
- seguridad reforzada,
- exportaciones avanzadas,
- multiplataforma completa.

## Uso de esta matriz

Esta matriz debe utilizarse para:

1. comprobar que una tarea responde a una necesidad real del producto,
2. evitar construir funcionalidades fuera del alcance acordado,
3. validar cobertura del MVP antes de iniciar Fase 4.
